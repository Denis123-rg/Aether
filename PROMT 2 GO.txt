General Guidelines
Do not change production code – only write tests. If a function is unexported and hard to test, you may add a test-only exported wrapper or use //go:build test files, but prefer testing through exported APIs.

Use table‑driven tests where appropriate to cover multiple scenarios concisely.

Mock external dependencies – use interfaces and dependency injection to replace real RPC clients, databases, and signers with test doubles. Use gomock or testify/mock as needed.

Test error paths – many uncovered branches are error handling (e.g., database failures, network timeouts). Simulate these errors using mocks.

Zero‑coverage functions: For main() functions (e.g., in cmd/executor/main.go, deploy/docker/mock-builder/main.go), consider refactoring the logic into a separate function that can be tested (e.g., runMain()). The test can then call that function with controlled arguments and verify behaviour.

Generated code: For internal/pb, the ProtoMessage methods are trivial; they can be tested by simply creating a message and calling ProtoReflect().Descriptor() or by using the generated code in an integration test that marshals/unmarshals.

All new tests must pass – no existing tests should break.

Sub‑agent division: Assign each package (or set of related packages) to a sub‑agent. They should share mocks and test utilities to avoid duplication.

Specific Recommendations
cmd/executor: Many functions interact with external services (Flashbots, remote signer, admin endpoints). Use httptest, gRPC test servers, and mock clients to simulate all responses.

internal/db: The NoopLedger methods (InsertBundle, etc.) are trivial – add tests that call them and verify they return nil/do nothing. For NewPgLedger, test both success and failure cases (invalid DSN, connection errors).

internal/events: Test publisher/subscriber with mock Redis or use in‑memory substitutes.

internal/grpc: Test DialWithOptions with invalid addresses, TLS misconfigurations.

internal/risk: Test state transitions, circuit breaker, and daily reset logic with mocked time.

internal/signer: Test encryption/decryption with known key pairs, and error handling when files are missing or corrupted.

Deliverables
New test files (or additions to existing *_test.go files) that cover the listed functions.

Updated coverage report – run go test -coverprofile=cover.out ./... and go tool cover -func=cover.out to verify that each function now meets the ≥95% target.

Recommendations for future coverage maintenance.
