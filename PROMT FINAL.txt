# Final Code Quality Validation and Comprehensive Test Verification

## Objective

Your task is to perform the **final code quality validation** and **complete verification** of the project before moving on to the next development phase.

This is **NOT** a refactoring task.

This is **NOT** an architecture redesign.

This is **NOT** a feature implementation task.

Your goal is to make the project completely clean while preserving **100% of the existing behavior, architecture, APIs, protocol logic, consensus logic, networking logic, and execution flow.**

---

# General Requirements

Follow these rules throughout the entire process:

* Do **NOT** redesign the architecture.
* Do **NOT** refactor large portions of the code.
* Do **NOT** change public APIs unless absolutely required.
* Do **NOT** modify consensus logic.
* Do **NOT** modify execution logic.
* Do **NOT** change blockchain behavior.
* Do **NOT** change networking behavior.
* Do **NOT** change storage behavior.
* Do **NOT** introduce new features.
* Do **NOT** optimize unrelated code.

Only perform **minimal, safe, localized fixes** required to eliminate warnings or failing tests.

If multiple solutions exist, always choose the one with the **smallest possible code change**.

---

# Phase 1 — Static Analysis Cleanup

Run every available static analysis tool and carefully eliminate all reported issues.

## Rust

Run:

```bash
cargo clippy --all-targets --all-features -- -D warnings
```

Goal:

* Zero warnings
* Zero errors
* No behavior changes

Only make minimal code changes required by Clippy.

---

## Go Vet

Run:

```bash
go vet ./...
```

Resolve every reported issue carefully.

Pay special attention to:

* context leaks
* resource cleanup
* goroutine safety
* nil checks
* unreachable code
* suspicious constructs

Do not change application logic.

---

## Staticcheck

Run:

```bash
staticcheck ./...
```

Resolve every warning carefully.

Typical examples include:

* deprecated APIs
* unused test helpers
* unreachable code
* ineffective assignments
* unnecessary allocations
* simplifications that do not affect behavior

If a deprecated API must be updated, migrate to the recommended replacement while preserving identical behavior.

# Phase 2 — Build Verification


## Go

Compile every Go module/package.

Everything must build successfully.

# Phase 3 — Complete Test Suite

Run **every available test**.

Nothing should be skipped.

## Rust Tests

Run all Rust tests.

Examples:

```bash

```

Requirements:

* All tests pass
* No panics
* No compilation failures
* No ignored failures

---

## Go Tests

Run all Go tests.

Example:

```bash
go test ./...
```

Example:

```bash

```

Requirements:

* All tests pass
* No failures
* No race conditions (if race tests are available)
* Coverage is successfully generated
